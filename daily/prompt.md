## Prompt Builder模块设计

Prompt Builder 是智能题目生成模块中的核心组件，其主要职责是根据用户输入、课程知识库检索结果以及业务规则，动态构建发送给大语言模型的 Prompt。在整个系统中，Prompt Builder 并不是简单地拼接字符串，而是负责将业务需求转换为大模型能够理解的自然语言指令，从而保证题目生成结果的准确性、稳定性和一致性。


在最初的实现方案中，我直接将用户输入的知识点发送给大模型，比如：请生成 5 道关于 AVL 树的选择题。

这种方式虽然能够生成题目，但在实际测试过程中存在以下问题：

输出格式不固定，影响前端解析。

不同知识点生成质量差异较大。

容易生成教材之外的内容。

难度无法控制。

无法保证一定返回答案和解析。

Prompt Builder 接收多个模块的数据，而不仅仅是学生输入。

| 参数 | 来源 | 示例 |
|------|------|------|
| 课程名称 | 前端 | 数据结构 |
| 知识点 | 学生输入 | AVL树 |
| 题型 | 前端 | 选择题 |
| 难度 | 系统 | 中等 |
| 题目数量 | 系统 | 5 |
| 教材内容 | Retriever | AVL树相关知识 |

其中，教材内容由 Retriever 从知识库中检索获得，作为 Prompt 的上下文信息。


## Prompt 构建流程

学生输入知识点->检索器检索教材内容->Prompt Builder->构建完整 Prompt->调用大模型->返回 JSON 格式题目

Prompt Builder 位于知识检索与大模型之间，负责连接业务逻辑和模型推理过程。


**prompt组成**

1. Role

定义模型身份。

比如：你是一名具有丰富教学经验的数据结构教师。

作用：保持回答风格一致、保证内容符合教学场景。


2.Task（任务）

明确告诉模型需要完成什么任务。

比如：根据 AVL 树知识生成 5 道中等难度选择题。

作用： 明确生成目标、减少模型理解偏差。

3.Context（上下文）

由 Retriever 检索教材内容作为参考。

比如：AVL 树是一种平衡二叉搜索树，任意节点左右子树高度差不超过 1，可通过左旋和右旋保持平衡。

作用：减少模型幻觉、保证题目符合课程内容。

4.Constraint（约束）

限制模型输出。

比如：必须包含答案、必须包含解析、不允许生成教材之外内容、难度为中等、不允许重复题目。

作用：提高输出稳定性。

5.Output Format（输出格式）

统一规定返回 JSON。

比如：

```json
{
    "question":"",
    "options":[],
    "answer":"",
    "analysis":"",
    "knowledge_point":"",
    "difficulty":""
}
```

作用：方便前端直接解析，提高接口稳定性。


**Prompt Builder 输出示例**

最终发送给大模型的 Prompt 类似如下：

```
你是一名大学数据结构教师。

请根据下面教材内容生成 5 道中等难度选择题。

教材内容：

AVL 树是一种平衡二叉搜索树......

要求：

1. 必须包含答案；
2. 必须包含解析；
3. 不允许超出教材内容；
4. 输出 JSON 格式。

返回字段：

question
options
answer
analysis
knowledge_point
difficulty
```

**后续优化方向**

下一阶段计划继续优化 Prompt Builder，包括：

（1）Prompt 模板管理

不同题型使用不同 Prompt 模板，比如：

选择题模板
判断题模板
填空题模板
简答题模板

减少重复开发，提高维护效率。

（2）Few-shot Prompt

在 Prompt 中加入标准示例，引导模型输出更加稳定的结果。


（3）动态难度控制

根据用户选择的难度自动调整 Prompt，例如：

简单：基础概念理解。
中等：知识综合应用。
困难：算法分析与推导。

（4）Prompt Version 管理

建立 Prompt 版本管理机制，记录不同版本 Prompt 的修改内容及生成效果，持续优化题目质量。
